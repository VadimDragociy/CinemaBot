## Про самого бота

1) из библиотек я использовал aiohttp и aiogramm (альтернативы не пробовал ибо эти работают норм), а также aiosqlite
2) В ходе исследования я обнаружил две вещи:
    - мой ноутбук не может поддержать нормальную базу данных с фильмами
    
        ну то есть как бы может, но это по факту работает гораздо медленнее, чем запросы к api, поэтому локально у меня хранится только база данных sqlite с историей запросов пользователей

    - в вк и на рутубе столько пиратских фильмов, что просто уму непостижимо
    
        поэтому я принял волевое решение делать запросы только в вк и на рутуб ввиду ограниченности во времени. Однако у рутуба нет открытой документации к Api??? хотя апи есть, только надо им на почту писать, лол
        ну в общем сейчас поиск ведется только среди видосиков с вк

3) бот работает следующим образом - делается запрос по апи на [неофициальное апи кинопоиска](https://api.poiskkino.dev/documentation#/%D0%A4%D0%B8%D0%BB%D1%8C%D0%BC%D1%8B%2C%20%D1%81%D0%B5%D1%80%D0%B8%D0%B0%D0%BB%D1%8B%2C%20%D0%B8%20%D1%82.%D0%B4./MovieController_searchMovieV1_4)

    Оттуда же я беру рейтинг, постер и описание

    Если фильм находится там по запросу пользователя, то все шикарно, идем в вк и там по запросу user_query + фильм ищем наиболее релевантные ссылки (апи вконтакта клянется, что они наиболее релевантные)

    Если фильма на кинопоиске нет, то увы и ах
